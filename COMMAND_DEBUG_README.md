# 命令调试功能说明

## 功能概述

为测试客户端添加了完整的命令用法说明和调试功能，便于开发者和用户快速了解和测试系统提供的各种API命令。

## 新增文件

### 1. `command_help.md`
完整详细的命令帮助文档，包含：
- 所有可用命令的详细说明
- 请求/响应数据格式
- 使用场景示例
- 错误处理指南
- 完整的调试建议
- 代码示例

### 2. `test_commands.py`
命令快速测试脚本，提供：
- 自动化测试注册、登录功能
- 获取任务列表和定义
- 获取用户信息
- 检查版本
- 结果验证和错误处理

### 3. `managers/settings_manager_gui.py` (已更新)
在设置页面添加了：
- 📖 命令帮助按钮 - 点击可打开完整的命令帮助文档
- 支持复制帮助内容到剪贴板

## 使用方法

### 方法一：通过GUI查看帮助

1. 启动测试客户端
2. 切换到 **设置** 标签页
3. 在页面顶部找到 **📖 打开命令帮助文档** 按钮
4. 点击按钮即可查看完整的命令帮助文档
5. 可以点击 **📋 复制全部内容** 按钮将文档复制到剪贴板

### 方法二：直接查看Markdown文档

打开 `client_debug/command_help.md` 文件，使用任何Markdown查看器或文本编辑器查看。

### 方法三：运行测试脚本

```bash
# 在 client_debug 目录下运行
python test_commands.py
```

测试脚本将自动执行以下操作：
1. 创建测试用户账号
2. 登录系统
3. 获取任务列表
4. 获取任务定义
5. 获取用户信息
6. 检查客户端版本

## 命令列表概览

### 认证类命令 (2个)
- `register` - 用户注册
- `login` - 用户登录

### 任务管理类命令 (3个)
- `get_default_tasks` - 获取所有任务列表
- `get_task_definition` - 获取单个任务定义
- `sync_all_tasks_definitions` - 批量同步任务定义

### 图像处理类命令 (2个)
- `process_image` - 核心功能：处理屏幕截图并返回触控指令
- `process_queued_request` - 处理排队的请求

### 用户信息类命令 (1个)
- `get_user_info` - 获取用户详细信息和配额状态

### 版本检查类命令 (1个)
- `check_version` - 检查最新客户端版本

### 运行中操作管理类命令 (3个)
- `get_running_operations` - 获取当前运行中的操作列表
- `cancel_operation` - 取消指定操作
- `update_operation_params` - 更新操作参数

## 调试技巧

### 1. 查看日志
所有命令执行都会记录到日志文件：
- 位置: `client_debug/logs/`
- 过滤关键词: `COMMUNICATION` 可以查看所有通信相关日志

### 2. 使用重连机制
登录成功后，客户端会自动启用重连机制：
- 最多重试 3 次
- 每次间隔 4 秒
- 仅在登录状态下启用

### 3. 自定义超时时间
```python
communicator = ClientCommunicator(
    host="127.0.0.1",
    port=9999,
    timeout=600  # 10分钟（默认300秒）
)
```

### 4. 错误处理
常见错误类型已在帮助文档中详细列出，包括：
- `session_expired` - 会话过期
- `provider_rate_limited` - 供应商限流
- `quota_exceeded` - 配额不足
- 等等...

## 注意事项

1. **必须先登录** - 除注册、登录、版本检查外的所有命令都需要有效的 session_id
2. **配额限制** - 每个用户每天有使用次数限制
3. **图像大小** - 建议控制屏幕截图在 1024 像素以内
4. **并发限制** - 不建议同一用户同时发起多个 process_image 请求

## 技术支持

如果在使用过程中遇到问题：
1. 查看完整的 `command_help.md` 文档
2. 检查日志文件获取详细错误信息
3. 运行 `test_commands.py` 测试基础功能是否正常
4. 确认服务端正常运行并监听正确的端口
