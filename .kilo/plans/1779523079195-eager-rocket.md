# Directory Structure Refactoring Plan

## Current Issues

1. **Chinese directory names** - `入口/`, `安卓相关/`, `PC 端服务/` make navigation and imports difficult
2. **Root directory clutter** - 17 top-level items including temp/cache dirs
3. **Redundant config locations** - Config exists in root `config/` AND `入口/GUI/config/` AND `入口/config/`
4. **Unclear module boundaries** - `入口/` contains both GUI code and CLI tools
5. **Test files scattered** - Tests in root `tests/` but also in `入口/GUI/pyqt_ui/test_*.py`

## Temp/Cache Directories (exclude from version control)

These should remain in `.gitignore` only:
- `cache/`
- `logs/`
- `debug/`
- `debug_output/`
- `extracted_images/`
- `data/` (runtime data)
- `__pycache__/`
- `.pytest_cache/`

## Migration Steps

### Phase 1: Create New Structure
1. Create `src/` directory with subdirectories
2. Move `安卓相关/` → `src/core/`, `src/device/`, `src/screenshot/`
3. Move `入口/` → `src/gui/`
4. Consolidate all configs to root `config/`
5. Move assets to `assets/`

### Phase 2: Update Imports
All files need import path updates:
- Old: `from core.logger import ...`
- New: `from src.core.logger import ...`

Or use package-relative imports with `pyproject.toml` configured properly.

### Phase 3: Update Entry Points
- `src/gui/tkinter/main.py` - Tkinter entry
- `src/gui/pyqt6/main.py` - PyQt6 entry
- Update any launch scripts

### Phase 4: Clean Up
- Remove old directories after verification
- Update `AGENTS.md` with new structure
- Add `pyproject.toml` for proper Python package setup

## Benefits

1. **English directory names** - Universal accessibility
2. **Clean root** - Only essential items visible
3. **Clear separation** - `src/` for code, `tests/` for tests, `config/` for configs
4. **Standard layout** - Follows Python best practices
5. **Better imports** - Consistent import paths throughout

## Tradeoffs

**Pros:**
- Much clearer structure
- Easier for agents/new developers to navigate
- Follows Python packaging conventions
- Single config location reduces confusion

**Cons:**
- Breaking change - all imports must be updated
- Requires careful testing after migration
- Existing scripts/references will need updates

## Recommendation

**APPROVED: Full Migration** - User has confirmed they want the complete English rename with src/ structure.

Proceed with this refactoring. The current Chinese directory names are a significant barrier to maintainability. The proposed structure follows Python best practices and will significantly improve code discoverability.
